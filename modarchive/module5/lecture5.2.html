User Research

How do we learn about how users want the information to flow?
  • Literature Review

    Learning from prior work. Begin with a literature review to see if you can learn from the work of social scientist, anthropologists, domain experts, and others who may work with the user groups you are interested in.


  • Interviews*

    Guided 1-1 conversation. Identify your list of topics. Create a list of questions that you'd like to cover. Record, transcribe, or have a note-taker for the conversation to capture key lessons: challenges, needs, desires, attitudes, experiences, etc. Sampling method (inclusion criteria etc.)

* If done in the context of research, IRB is likely required.

Example: Interview

AXILAB Google Drive: Resilient Foods Interview. AXILAB Miro: Software Group Pilot Interviews

User Research (cont)

  • Focus Groups*

    Moderated group discussion. Create a set of topics to cover, establish ground-rules for group (if any). Record, transcribe, or have a note-taker for the conversation to summarize key lessons: challenges, needs, desires, attitudes, experiences, etc. Group construction is a key detail (who, how many, backgrounds, etc).

  • Task Analysis*

    Observing a user performing daily tasks to meet their intended goals. This method allows us to learn about how and why people use specific technologies.

  • Decision Mapping*

    Discussion mapping how a user collects, structures, manages, uses, and thinks about data to drive decision making.

* If done in the context of research, IRB is likely required.

UI Inventory

  • Assets

    Any logos, images, videos, and other media to use in the UI. If you have collections of images, consider consistent naming and folder structure for each inventorying.

  • Palette

    Determine some basic colors: background, a color pop, normal font, "highlight" font.

    • Random palette generator: Coolors.co
    • Dig deeper into color theory: Adobe Color Tool
    • Test your color palette for visual appeal and accessibility:Material Color UI Tool

  • Typography

    Pick a set of consistent fonts and font sizes. When starting out, consider just using 1-2 font types and stick with 4ish sizes (header, subheader, main text, small text). Learn more about typography for UI design here.

  • Pattern Library

    Create a small collection of patterns for your UI components (buttons, cards, etc.) You can learn more aboutUI Design Patterns here. You can also use an open-source UI toolkit, like Bootstrap to guide your design.

User Interface Design

  1. Start with your user flow
    • Know the overarching user workflow from one page to the next
  2. Atomize your Design
    • Determine smallest UI units and how they combine into larger components
  3. Create a UI Inventory
    • Determine a consistent look and feel for all UI units
  4. Find a Toolkit
    • Pick a UI Library that works for you, e.g., Bootstrap.
  5. Mockup, Test, Build, Test
    • Create a mockup with enough fidelity to guide what you build
    • Perform user testing with your mockups to catch bugs early
    • Build and test your code! Rinse and refine!
